# Анимированный шрифт + рантайм: идеи и реализация

**Рабочее название экосистемы (шрифт + манифест + рантайм) в будущем:** *Dynamic font* — отдельно от репозитория **Mops Font Generate**; здесь описаны идеи, которые могут лечь в основу продуктовой линии *Dynamic font* или встроенных демо в Mops.

Документ фиксирует обсуждённые концепции: **почему шрифт сам по себе не «анимируется» везде**, как сделать **простое внедрение**, стили вроде **Anim Bold**, **панель «осей» для движения**, технические приёмы для эффектов и **что можно добавить в Mops**.

---

## 1. Почему не хватает одного файла `.woff2`

- Движок текста (браузер, Word, Figma) рисует глифы как **статичную геометрию** (контур / слои COLR и т.д.), без сценария «кино внутри глифа» для любого приложения.
- **Вариативные шрифты** тоже изначально требовали **дополнительного слоя**: CSS (`font-variation-settings`), иногда JS для слайдеров — это уже стала **норма**, а не «костыль».
- **Аналогия с Rive**: интерактив в **`.riv`**, на сайте — **JS + WebAssembly runtime** ([документация Rive для Web](https://rive.app/docs/runtimes/web/web-js)). Без рантайма файл не оживает сам по себе.

**Вывод:** модель **«шрифт + небольшой пакет (манифест + рантайм)»** — осознанный и распространяемый паттерн для веба; в универсальных редакторах без вашего кода останется только статичный глиф.

---

## 2. Цель продукта: простое внедрение

**Минимальный контракт для автора сайта**

1. Подключить **один бандл** (ESM или `<script type="module">` с CDN).
2. Вызвать **одну инициализацию**: контейнер / поле ввода, URL шрифта, URL **манифеста** (JSON) или встроенный объект настроек.

По смыслу:

```text
AnimatedFont.mount({ root: '#editor', fontUrl: '…woff2', themeUrl: '…json' })
```

**Что поставляет дизайнер шрифта**

| Файл | Назначение |
|------|------------|
| `MyFont.woff2` | Обычное начертание |
| `MyFont.anim.json` (или `theme.json`) | Стили **Anim Bold**, пресеты движения, привязка к осям VF, тайминги |

Итого: **два файла + одна строка кода** — низкий порог входа.

## 3. Стили «Anim Bold» рядом с Regular / Bold

**Идея:** в манифесте секция **`styles`**, по аналогии с CSS, но для **движения** и опционально **VF**:

- `Regular` / `Bold` — как обычно: `font-weight`, при необходимости оси `wght` / др.
- **`Anim Bold`**, **`Anim Regular`** — тот же визуальный вес + **набор параметров анимации** (появление, ластик, stagger и т.д.).

На сайте в выпадающем списке — не только 400/700, но и **Anim …**; рантайм вешает классы / CSS variables / оси.

**Альтернатива — одна виртуальная «ось» в UI**

- Ось `ANIM` 0–100: 0 = выкл, больше = сильнее эффект.  
- Ближе к привычной **строке осей вариативного шрифта** — без размножения имён стилей.

**Рекомендация:** в продукте совместить **пресеты** (для новичков) и **слайдеры Motion** (для тонкой настройки).

---

## 4. «Строка осей» для анимации (метафора VF)

Отдельная группа контролов **Motion** (рядом с `wght` / `wdth`):

| Условная «ось» | Смысл |
|----------------|--------|
| **IN** | интенсивность появления (0–100) |
| **DEL** | задержка между глифами (stagger) |
| **WIP** | «ластик»: скорость / ширина штриха |
| **HOP** | подпрыгивание / overshoot |

Опционально **`vfBridge`** в манифесте: реальная ось VF → параметр движения (например, ширина маски «ластика» от `wdth`).

## 5. Технические реализации эффектов (веб)

### 5.1 Появление снизу для каждого глифа

- **CSS:** символы в `inline-block`, `transform: translateY(…) → 0`, `opacity`, **stagger** через `animation-delay` по индексу.
- **Variable font:** кастомная ось в шрифте (например вертикальный сдвиг метрик) + анимация `font-variation-settings` — «движение родное контуру», но ось нужно заложить при дизайне шрифта.
- **Canvas / SVG:** сложнее, но свобода (тени, bounce, частицы).

### 5.2 «Ластик»

Два разных чтения — оба реализуемы:

1. **Стирание:** движущаяся полоса маскирует букву (`clip-path`, `mask-image`, canvas).
2. **Появление:** поверх буквы слой цвета фона; «ластик» снимает маску (revealing).

**Связь со шрифтом:** форму задаёт шрифт; «ластик» — **оверлей / маска**, не таблица внутри OTF.

### 5.3 Лигатуры и сложные скрипты

- Анимировать «по одному `char`» ломает связки → использовать **графемные кластеры** (`Intl.Segmenter` или библиотека).

### 5.4 Доступность и UX

- Не ломать копирование и чтение скринридерами: по возможности **видимый текст в DOM** + декоративный слой.
- На длинных текстах — **упрощать** эффект или батчить обновления.

## 6. Схема манифеста (черновик v0.1)

Иллюстративная структура JSON (поля можно сузить под первый релиз):

```json
{
  "version": 1,
  "fontFamily": "MyAnimated",
  "styles": {
    "Regular": { "fontWeight": 400, "motion": { "preset": "none" } },
    "Bold": { "fontWeight": 700, "motion": { "preset": "none" } },
    "Anim Bold": {
      "fontWeight": 700,
      "motion": { "preset": "rise", "durationMs": 220, "staggerMs": 24, "easing": "ease-out" }
    }
  },
  "presets": {
    "rise": { "type": "translateY", "from": "0.35em", "opacity": [0, 1] },
    "eraser-in": { "type": "mask-reveal", "direction": "ltr" }
  },
  "vfBridge": {
    "wdth": { "motionParam": "eraserWidth", "scale": [0.5, 1.5] }
  }
}
```

Рантайм читает манифест и применяет классы / переменные / оси.

## 8. Связь с текущей кодовой базой (Mops)

| Область | Файлы / заметки |
|---------|------------------|
| Статическая генерация из VF | `pages/api/generate-static-font.js` — Python `fonttools` локально, на Vercel — `@web-alchemy/fonttools` + Pyodide; временные файлы в `os.tmpdir()` |
| Трейсинг Pyodide на Vercel | `next.config.js` → `experimental.outputFileTracingIncludes` для `/api/generate-static-font` |
| Лимиты serverless | `vercel.json` + `maxDuration` для долгой первой загрузки WASM |

Идея **отдельного MD про motion-пакет** не заменяет `FONT_GENERATION_ROADMAP.md` — там про генерацию шрифтов; здесь про **продукт «шрифт + рантайм»** и UX.

---

## 9. Что сознательно не обещать в первой версии рантайма

- Работа «везде как системный шрифт» без кода (Word, случайный CMS).
- Идентичный вид во всех браузерах без тестов маски / `clip-path`.
- Нулевая стоимость по байтам: Pyodide / canvas — осознанный компромисс.
